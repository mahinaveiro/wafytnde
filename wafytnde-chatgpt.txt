You are building a complete offline-first PWA named “Wafytnde”.

This is a personal note-taking, idea-capturing, project-planning, bundle-organizing web app. It must feel like a lightweight desktop utility from an alternate retro Windows XP / early Linux / old paper UI universe, but polished with modern interaction quality. It must not look like a generic AI-generated SaaS website.

The app’s purpose:
A user suddenly gets ideas and needs to capture them instantly without friction. The app should open fast, work offline, store everything locally, and let the user organize thoughts into bundles, projects, notes, todo lists, archives, and quick captures.

Do not create a marketing website. Do not create a landing page. Build the actual app.

Core priorities:
1. Extremely fast.
2. Offline-first PWA.
3. Fully responsive, especially phone-first.
4. Local data storage.
5. Beautiful retro desktop-inspired UI.
6. No emojis anywhere.
7. No generic AI website aesthetics.
8. No bloat.
9. No server backend.
10. No account system.
11. Import/export data must work reliably between desktop and phone.

Tech direction:
Use the existing project stack if one already exists. If starting fresh, prefer Vite + React + TypeScript. Use a simple state system, not over-engineered architecture. Use IndexedDB for all main app data. Use localStorage only for tiny preferences such as theme, last opened workspace, sidebar collapsed state, layout settings, and onboarding completion. Do not store the entire note database in localStorage.

Storage:
Use IndexedDB through a clean abstraction layer.
Create a versioned database schema.
Support migrations.
Data must survive reloads and offline usage.
Every record must include:
- id
- createdAt
- updatedAt
- archivedAt if archived
- deletedAt if soft deleted
- order/index for manual sorting where needed
- type where needed
- parent references where needed

Import/export:
This is critical.
Export must gather all IndexedDB data plus localStorage preferences into one downloadable backup file.

Export format:
Use a JSON file with clear versioning.
Filename format:
wafytnde-backup-YYYY-MM-DD-HH-mm.json

Export object structure:
{
  "app": "Wafytnde",
  "schemaVersion": number,
  "exportedAt": ISO string,
  "data": {
    "bundles": [],
    "projects": [],
    "notes": [],
    "todoLists": [],
    "todoItems": [],
    "quickCaptures": [],
    "settings": {}
  }
}

Import behavior:
- User selects backup JSON file
- Validate file before importing
- Check app name and schema version
- Show preview:
  - number of bundles
  - number of projects
  - number of notes
  - number of todos
  - exported date
- Give options:
  - Merge with current data
  - Replace all current data
- Merge must avoid duplicate IDs
- If duplicate IDs exist, preserve incoming relationships by remapping IDs carefully
- Replace must ask for strong confirmation
- After import, show a clean success panel
- If import fails, show clear error without destroying existing data
